# What Users Do on the Platform

This document explains, in simple words, what each type of user does on the goal-setting and tracking portal.

---

## 1. Employee

An **Employee** uses the portal to set their own goals and regularly update progress.

### 1.1 At the start of the year

- Logs into the portal.
- Creates a **Goal Sheet** for the current year (cycle).
- Adds a set of clear goals (up to 8), for example:
  - “Increase sales by 15%”
  - “Finish Project X by December”
- For each goal, chooses:
  - A **Thrust Area** (which big theme it belongs to).
  - A **Unit of Measurement (UoM)**:
    - Number (e.g., 100 calls)
    - Percentage (e.g., 95% quality)
    - Timeline (finish by a date)
    - Zero-based (e.g., 0 safety incidents)
  - A **Target** (what success looks like).
  - A **Weightage** (how important this goal is out of 100%).

When all goals are set and the total weightage = 100%, the employee **submits** the goal sheet to their manager for approval.

### 1.2 During the year (quarterly)

Each quarter (Q1, Q2, Q3, Q4):

- Opens the **Quarterly Check-in** page.
- For each goal, fills in:
  - What they actually achieved in that quarter.
  - Status: **Not Started**, **On Track**, or **Completed**.
  - Optional comments or notes.

The system shows simple progress scores and whether the employee is on track against the targets.

### 1.3 For shared/departmental goals

- Some goals may be **shared** with many people in the team or department.
- For these shared goals:
  - The title and target are fixed by the manager/admin.
  - The employee can only adjust the **weightage**.
  - Progress may auto-update if the primary owner updates it.

**In short:**  
An employee **sets goals once** at the start of the cycle and **updates progress every quarter**.

---

## 2. Manager (L1)

A **Manager** uses the portal to review and approve goals, then hold structured check-ins with their team.

### 2.1 Goal review and approval

At the start of the cycle:

- Opens the **Team Goal Sheets** view.
- For each team member:
  - Reads the set of goals they submitted.
  - Adjusts **targets** or **weightages** where needed (inline editing).
  - Either:
    - **Approves** the goal sheet, or
    - **Sends it back for rework** with comments.

Once a goal sheet is approved:

- The sheet and its goals become **locked**.
- Employees can no longer edit those goals unless an admin unlocks them.

### 2.2 Quarterly check-ins

Each quarter:

- Opens the **Check-in Dashboard** for their team.
- For each team member:
  - Sees **Planned vs Actual** values for every goal.
  - Reviews the status (Not Started / On Track / Completed).
  - Adds a **check-in comment** summarizing the discussion and feedback.

When a check-in is done for a person in that quarter, the manager marks it as **completed** so Admin/HR can see that the loop is closed.

### 2.3 Shared and team-level goals

- Can push **shared goals** / KPIs to multiple team members (with help from Admin if needed).
- Uses the dashboard to see how the team is performing on these shared goals.

**In short:**  
A manager **approves goals once** at the start, and **reviews and discusses progress every quarter**.

---

## 3. Admin / HR

An **Admin/HR** user configures the system, controls timelines, and monitors overall completion and data quality.

### 3.1 Setup and configuration

- Creates **review cycles** (for example, “FY2026”).
- For each cycle, sets the **windows**:
  - When goal setting is open.
  - When Q1, Q2, Q3, and Q4 check-ins are open.
- Manages the **organization hierarchy**:
  - Which employee reports to which manager.
  - Which users are admins.

### 3.2 Monitoring and governance

- Uses the **Completion Dashboard** to check:
  - Which employees have created and submitted goals.
  - Which managers have finished quarterly check-ins.
- Uses **reports** (CSV/Excel) to see:
  - Planned targets vs actual achievements across the organization.
  - Progress at employee, team, and department level.
- Reviews the **Audit Log** to see:
  - Who changed what, and when, especially after goals were locked.

### 3.3 Exceptions and support

- Can **unlock** a goal sheet or specific goals if something must be corrected after approval.
- Can help create department-wide **shared goals** that appear for many employees.
- (Optional/advanced) Can configure **notifications** and **escalation rules** so:
  - People get reminders to submit goals and check-ins.
  - Escalations happen if someone delays too long.

**In short:**  
Admin/HR **sets up the rules and periods**, **monitors completion and reports**, and **handles exceptions and governance**.

---

## 4. One-line Summary per Role

- **Employee:** “I define my goals and update my progress each quarter.”
- **Manager:** “I approve my team’s goals and review their progress every quarter.”
- **Admin/HR:** “I set up the cycles, windows, and hierarchy, and make sure everyone follows the process.”
